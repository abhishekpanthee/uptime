## Abstract

The Uptime Tracker is a web-based monitoring system designed to track website availability and performance metrics in real-time. Similar to platforms like status.tcioe.edu.np and downdetector.com, this system provides automated monitoring of multiple websites, tracking their uptime status, response times, and historical availability data. The system features user authentication, website registration, periodic ping checks every 5 minutes, and comprehensive analytics dashboards. Using PostgreSQL for data storage and automated monitoring scripts, the platform enables both public and private monitoring of websites, making it ideal for educational institutions and organizations to ensure service reliability.

## 1. Introduction

### 1.1 Background

In today's digital era, website availability is critical for businesses and educational institutions. Websites serve as primary interfaces for information dissemination and service delivery. Monitoring these services manually is inefficient and time-consuming. Automated uptime monitoring systems provide real-time status information and historical analytics for better service management.

### 1.2 Motivation

Educational institutions like Thapathali Campus rely on multiple web services. When these services experience downtime, users are often unaware of the issue's scope or duration. A centralized monitoring system can provide transparency, enable quick response to incidents, and maintain historical data for performance analysis.

### 1.3 Problem Statement

Currently, there is no centralized system to monitor college websites and services. This leads to delayed response to downtime, lack of transparency, no historical performance data, and poor user experience during service disruptions.
