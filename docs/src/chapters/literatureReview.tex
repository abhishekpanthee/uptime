\chapter{LITERATURE REVIEW}

\section{Introduction to Website Monitoring}

Website monitoring involves automated systems that periodically test whether websites are accessible using \gls{http}/\gls{https} protocols \cite{sommerville2016software}. Modern monitoring tracks availability, response time, and \gls{http} status codes to help administrators measure service quality \cite{berners1996http}.

\section{Importance of Uptime Monitoring}

Website downtime carries significant costs---research indicates IT downtime costs organizations an average of \$5,600 per minute \cite{ponemon2016cost}. For educational institutions, downtime disrupts learning activities and access to critical resources. Proactive monitoring enables early detection, supporting \gls{sla} compliance and capacity planning \cite{gartner2014downtime}.

\section{Existing Monitoring Solutions}

\subsection{Commercial Platforms}

\textbf{Pingdom} \cite{pingdom2023} offers comprehensive multi-location monitoring with synthetic transactions and analytics, though pricing scales with features. \textbf{UptimeRobot} \cite{uptimerobot2023} provides a freemium model with 5-minute checks on the free tier. \textbf{StatusCake} and \textbf{Datadog} offer additional options for different organizational needs.

\subsection{Open-Source Solutions}

\textbf{Uptime Kuma} is a popular self-hosted solution with a clean interface supporting HTTP, TCP, and DNS monitoring. \textbf{Nagios} and \textbf{Zabbix} offer enterprise-grade capabilities but require significant configuration expertise. \textbf{Prometheus with Blackbox Exporter} enables monitoring in cloud-native environments.

\section{Technical Approaches to Monitoring}

\subsection{HTTP-Based Monitoring}

HTTP monitoring sends requests to URLs and analyzes responses. Status codes indicate outcomes: 2xx codes indicate success, 4xx client errors, and 5xx server errors. Response time measurement captures total time from request to completion, providing performance insight beyond availability.

\subsection{Check Intervals and Data Aggregation}

A 5-minute interval balances detection speed with resource consumption, generating 288 data points per day per site. \textbf{Hourly aggregates} summarize checks into average response time and availability percentage. \textbf{Daily aggregates} enable long-term trend analysis while managing storage efficiently.

\section{Database and Technology Considerations}

Monitoring systems generate time-series data \cite{date2003introduction}. PostgreSQL handles moderate \gls{dbms} workloads effectively with temporal data types and \gls{sql} aggregation capabilities \cite{postgresql2024, codd1970relational}.

\end{itemize}
