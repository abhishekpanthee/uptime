\section{Website Monitoring and Availability}

Website availability monitoring is critical for modern web infrastructure, with IT downtime costing approximately \$5,600 per minute \cite{gartner2014}. Monitoring systems employ active probing---sending periodic HTTP requests to verify availability by checking response codes and response times \cite{rfc7231}.

Patterson et al. \cite{patterson2002} defined uptime as the percentage of time a system remains operational, typically expressed as "nines" (e.g., 99.9\% equals 8.76 hours downtime per year) in Service Level Agreements.

\section{Existing Monitoring Solutions}

\subsection{Commercial Platforms}

Pingdom offers comprehensive monitoring with global probe locations \cite{pingdom2023}. UptimeRobot provides 5-minute monitoring intervals with a freemium model \cite{uptimerobot2023}. Downdetector aggregates user reports rather than active monitoring \cite{downdetector2023}.

\subsection{Open-Source Solutions}

Uptime Kuma is a popular self-hosted solution with Docker deployment \cite{uptimekuma2023}. Nagios and Zabbix provide enterprise-grade capabilities but require significant expertise \cite{nagios2023, zabbix2023}. Prometheus with Blackbox Exporter enables HTTP probing within cloud-native environments \cite{prometheus2023}.

\section{Technical Approaches}

HTTP monitoring sends requests to URLs and analyzes response status codes, response time, and SSL certificate validity \cite{rfc7231}. ICMP ping provides basic reachability but doesn't verify application functionality \cite{rfc792}. Five-minute intervals balance detection speed with resource consumption \cite{mansmann2007}.

\section{Data Storage}

PostgreSQL handles time-series monitoring data efficiently with temporal queries and aggregations \cite{postgresql2023}. For moderate data volumes, it provides adequate performance with simpler deployment than specialized databases like InfluxDB \cite{influxdb2023}.

\section{Research Gaps}

Existing solutions have limitations for educational institutions:
\begin{itemize}
    \item Commercial solutions impose recurring subscription costs
    \item Enterprise tools (Nagios, Zabbix) require significant expertise
    \item Cloud-based monitoring cannot detect internal network issues
    \item Limited customization for institutional requirements
\end{itemize}
