\chapter{REQUIREMENTS ANALYSIS}
\label{chap:requirements}

\section{Functional Requirements}

\begin{itemize}
    \item Users can create accounts and log in securely.
    \item Users can add their websites to be monitored.
    \item The system checks each website every 5 minutes automatically.
    \item Each check records whether the site is up and how fast it responds.
    \item The system calculates average uptime per hour and per day.
    \item Anyone can view public status pages showing website health.
    \item Users see a dashboard with charts and uptime history.
    \item Users can choose to make their monitors public or private.
\end{itemize}

\section{Non-Functional Requirements}

\begin{itemize}
    \item \textbf{Speed:} Pages and API calls load in under 500 milliseconds.
    \item \textbf{Reliability:} The monitoring service runs 99\% of the time.
    \item \textbf{Capacity:} The system can handle 100+ websites being monitored.
    \item \textbf{Mobile-friendly:} The dashboard works well on phones and tablets.
    \item \textbf{Security:} User passwords are encrypted before storing.
\end{itemize}

\section{System Requirements}

\subsection{Hardware Requirements}
\begin{itemize}
    \item \textbf{Minimum:} Any 64-bit computer with 4 GB RAM and 10 GB storage
    \item \textbf{Recommended:} Dual-core processor, 8 GB RAM, SSD for faster performance
\end{itemize}

\subsection{Software Requirements}
\begin{itemize}
    \item Operating System: Windows, Linux, or macOS
    \item Backend: Bun.js (JavaScript runtime)
    \item Frontend: Next.js with React
    \item Database: PostgreSQL
\end{itemize}
