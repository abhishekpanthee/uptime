\chapter{SYSTEM ARCHITECTURE AND METHODOLOGY}

\section{Introduction}

This chapter presents the architectural design of the Uptime Tracker system, detailing the component structure, \gls{dbms} schema, and \gls{api} design. The architecture follows a three-tier pattern separating presentation, business logic, and data storage.

\section{System Architecture Overview}

The Uptime Tracker employs a modern web architecture with four components:

\begin{enumerate}
    \item \textbf{Frontend:} Next.js-based \gls{ui} with dashboards and status pages
    \item \textbf{Backend \gls{api}:} Elysia.js server with \gls{rest} endpoints
    \item \textbf{Database:} PostgreSQL storing users and monitoring data
    \item \textbf{Monitoring Service:} Background process executing \gls{http} checks
\end{enumerate}

\section{Monitoring Service}

The monitoring service executes the following steps:
\begin{itemize}
    \item Scheduler triggers checks every 5 minutes
    \item Retrieves URLs from ownership table
    \item Sends parallel \gls{http} requests
    \item Records response times in analytics table
    \item Updates hourly aggregates
\end{itemize}

\section{User Interface Design}

The frontend uses a component-based architecture with: Landing Page, Dashboard (website status grid), Website Detail (historical charts), Add Website form, Public Status Page, and Settings. The dashboard displays website cards with URL, status indicator, response time, and uptime percentage.

\section{Development Methodology}

The project follows iterative development with incremental features, regular integration, and prioritized backlog. Development phases:
\begin{itemize}
    \item Foundation---setup and authentication
    \item Core Features---monitoring service
    \item Interface---dashboard and visualizations
    \item Polish---testing and error handling
\end{itemize}

\section{Deployment}

The system supports single-server deployment with all components, or separate services for scaling. Requirements: Linux server with Bun, PostgreSQL access, and HTTPS configuration.
